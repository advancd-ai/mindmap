# 노드 에디터 모드 시스템

**작성일**: 2024-12-19  
**구현 완료일**: 2024-12-19  
**버전**: 2.0.0  
**상태**: Phase 1-4 완료 ✅ | Phase 4.5 계획됨 ⭐

## 📋 개요

마인드맵 노드의 편집 방식을 사용자 경험과 편집 편의성에 따라 세 가지 모드로 구분하여 제공합니다. 각 모드마다 적합한 편집 인터페이스를 제공하여 다양한 사용자 니즈를 충족시킵니다.

## 🎯 목표

1. **빠른 수정**: 간단한 텍스트 수정을 위한 인라인 편집 모드
2. **풍부한 편집**: 서식이 많은 콘텐츠를 위한 오버레이 편집 모드
3. **일관된 UX**: 모드 간 전환이 자연스럽고 직관적

## 📌 노드 에디터 모드 종류

### 1. Text 모드 (인라인 편집)

**목적**: 빠른 텍스트 수정 및 간단한 편집  
**편집 위치**: 노드 위에 직접 표시되는 작은 텍스트 입력 필드

#### 특징
- ✅ 노드 위에서 직접 편집
- ✅ 최소한의 UI 간섭
- ✅ 빠른 입력 및 수정 가능
- ⚠️ 긴 텍스트 편집에 제한적
- ⚠️ 서식 지원 없음

#### 사용 사례
- 노드 라벨 간단 수정
- 빠른 텍스트 입력
- 단일 라인 텍스트 편집

#### UI/UX
```
┌─────────────────┐
│  [노드 영역]     │
│  ┌───────────┐  │
│  │ 텍스트 입력 │  │  ← 노드 위에 직접 표시
│  └───────────┘  │
│                 │
└─────────────────┘
```

#### 구현 요구사항
- `input` 또는 작은 `textarea` 사용
- 노드 크기 내에서만 표시
- 최대 높이 제한 (예: 3-5줄)
- 단축키: `Enter` 저장, `Esc` 취소

---

### 2. Rich Editor 모드 (오버레이 편집)

**목적**: 서식이 있는 콘텐츠를 편집하기 위한 풀스크린 편집 환경  
**편집 위치**: 화면 중앙에 표시되는 큰 오버레이 팝업

#### 특징
- ✅ 큰 편집 영역 제공 (전체 화면 또는 큰 모달)
- ✅ WYSIWYG 서식 편집 도구
- ✅ 복사/붙여넣기 지원 (HTML, 텍스트)
- ✅ 시각적 편집 인터페이스
- ⚠️ 노드에서 떨어져 있음 (컨텍스트 손실 가능)

#### 사용 사례
- 서식이 많은 텍스트 편집
- 리치 콘텐츠 작성
- 이미지, 링크 등 포함된 노드 편집
- 긴 텍스트 편집

#### UI/UX
```
┌─────────────────────────────────────────┐
│  [모달 오버레이 - 화면 중앙]              │
│  ┌───────────────────────────────────┐  │
│  │ [툴바: Bold, Italic, Link, ...]   │  │
│  ├───────────────────────────────────┤  │
│  │                                   │  │
│  │     넓은 편집 영역                  │  │
│  │     (최대 너비: 800px-1200px)      │  │
│  │     (최대 높이: 600px-800px)       │  │
│  │                                   │  │
│  │                                   │  │
│  ├───────────────────────────────────┤  │
│  │ [취소]              [저장]        │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
        배경 반투명 오버레이
```

#### 구현 요구사항
- 모달 컴포넌트 (`RichEditorModal.tsx`)
- `contentEditable` 기반 WYSIWYG 에디터
- 툴바: Bold, Italic, Underline, Link, Heading, List 등
- 단축키: `Alt+Enter` 저장 후 종료, `Esc` 취소
- DOMPurify를 통한 보안 처리
- 복사/붙여넣기 처리 (HTML sanitization)

#### 참고 컴포넌트
- `RichEditor.tsx`: 현재 인라인 에디터
- `RichEditorToolbar.tsx`: 툴바 컴포넌트
- 이를 기반으로 모달 버전 생성

### 3. Markdown Editor 모드 (오버레이 편집)

**목적**: 마크다운 문법으로 구조화된 텍스트 편집  
**편집 위치**: 화면 중앙에 표시되는 큰 오버레이 팝업 (Split View 또는 Live Preview)

#### 특징
- ✅ 큰 편집 영역 제공
- ✅ 마크다운 실시간 미리보기
- ✅ 마크다운 문법 하이라이팅 (선택사항)
- ✅ 코드 블록, 테이블 등 고급 문법 지원
- ⚠️ 마크다운 문법 학습 필요

#### 사용 사례
- 마크다운으로 구조화된 텍스트 작성
- 코드 블록이 포함된 문서
- 개발자 친화적 콘텐츠
- 버전 관리에 유리한 텍스트 포맷

#### UI/UX

**옵션 1: Split View (편집 + 미리보기)**
```
┌─────────────────────────────────────────┐
│  [모달 오버레이 - 화면 중앙]              │
│  ┌─────────────────┬─────────────────┐ │
│  │ [툴바: #, **, ] │ [미리보기 모드]  │ │
│  ├─────────────────┼─────────────────┤ │
│  │                 │                 │ │
│  │   마크다운       │   렌더된         │ │
│  │   편집 영역      │   미리보기        │ │
│  │                 │                 │ │
│  │   - Item 1      │   • Item 1     │ │
│  │   - Item 2      │   • Item 2     │ │
│  │                 │                 │ │
│  └─────────────────┴─────────────────┘ │
│  [취소]                    [저장]      │
└─────────────────────────────────────────┘
```

**옵션 2: Single View (편집만)**
```
┌─────────────────────────────────────────┐
│  [모달 오버레이 - 화면 중앙]              │
│  ┌─────────────────────────────────┐  │
│  │ [툴바: #, **, -, `, >]          │  │
│  ├─────────────────────────────────┤  │
```

#### 구현 요구사항
- 모달 컴포넌트 (`MarkdownEditorModal.tsx`)

## 🔄 모드 전환 및 선택

### 기본 편집 모드 설정

**현재 구현 상태**:
- 새 노드 생성 시: `contentType: 'richeditor'` 기본값 설정 ✅
- `contentType`이 `richeditor`인 경우: Rich Editor 모달 열림 ✅
- `contentType`이 미지정된 경우: Text 모드 (인라인) 편집 ✅

**계획된 개선 사항 (Phase 4.5)** ⭐:
- `contentType`이 미지정된 경우에도 Rich Editor 모달 열림
- 모든 새 노드가 Rich Editor로 편집 가능하도록 통일

### 노드 타입과 에디터 모드 매핑

| 노드 `contentType` | 기본 에디터 모드 | 대체 모드 |
|-------------------|----------------|----------|
| `text` (또는 미지정) | **Rich Editor** ⭐ | Text (인라인), Markdown (오버레이) |
| `richeditor` | **Rich Editor** ⭐ | Text (인라인), Markdown (오버레이) |
| `markdown` | Markdown Editor | Text (인라인), Rich Editor (오버레이) |

### 모드 전환 UI

#### 1. 도구 상자 (Canvas Toolbar) 아이콘 ⭐

**위치**: 화면 상단 도구 상자 (`canvas-toolbar`)

**기능**: 선택된 노드의 편집 모드를 빠르게 전환

```
┌─────────────────────────────────────────┐
│ [도구 상자]                              │
│ [Add Node] [Edit] [📝 Rich] [📄 Markdown]│
│                                         │
│ 📝 Rich: Rich Editor 모드로 편집 시작    │
│ 📄 Markdown: Markdown Editor 모드로 편집 시작 │
└─────────────────────────────────────────┘
```

**동작** (Phase 4.5 구현 예정):
- 노드 선택 시 도구 상자에 편집 모드 아이콘 표시
- 선택된 노드가 있을 때만 아이콘 표시
- 아이콘 클릭 시:
  1. 노드의 `contentType`을 해당 모드로 업데이트
  2. 해당 편집 모드로 즉시 편집 시작
- 현재 모드 표시: 활성 모드 아이콘 하이라이트

#### 2. 노드 헤더 버튼 (Text 모드일 때)

**위치**: Text 모드 노드의 헤더 영역 (선택된 노드만 표시)

```
┌─────────────────────────┐
│ [📝 Markdown] [✏️ Rich] │ ← 노드 헤더 (Text 모드일 때만 표시)
├─────────────────────────┤
│   텍스트 콘텐츠           │
└─────────────────────────┘
```

**동작**:
- Text 모드 노드가 선택되고 확장된 상태일 때만 표시
- 버튼 클릭 시:
  1. 노드의 `contentType`을 해당 모드로 업데이트
  2. 해당 편집 모드로 즉시 편집 시작

#### 3. 컨텍스트 메뉴 (향후 개선)

```
Right-click on node
  ├─ Edit
  │   ├─ Text Mode (인라인)
  │   ├─ Rich Editor Mode (오버레이)
  │   └─ Markdown Editor Mode (오버레이)
  └─ ...
```

### 기본 동작

**노드 클릭** (편집 버튼 표시):
- 노드를 클릭하면 선택됨
- 선택된 노드 위에 **"편집" 버튼** 표시 여부는 도구 상자의 "편집 버튼" 토글으로 제어 ✅
- **인라인 편집 모드일 때**: 편집 버튼은 기본적으로 숨김 처리 ✅
  - 도구 상자의 "편집 버튼" 버튼을 클릭하여 표시/숨김 토글 가능
- **일반 모드일 때**: 도구 상자의 "편집 버튼" 버튼으로 표시/숨김 제어
- 편집 버튼 클릭 시 → **모달 편집창** 열림 ✅
  - `contentType`이 `richeditor` → **RichEditorModal** 모달 열림
  - `contentType`이 `markdown` → **MarkdownEditorModal** 모달 열림
  - `contentType`이 `text` 또는 미지정 → **RichEditorModal** 모달 열림 (기본)

**노드 더블 클릭** (인라인 편집 - 팝업 없음):
- `contentType`이 `richeditor` → **Rich Editor** 인라인 편집 ✅
- `contentType`이 `markdown` → **Markdown Editor** 인라인 편집 ✅
- `contentType`이 `text` → Text 모드 (인라인) 편집 ✅
- `contentType`이 미지정 → Rich Editor 인라인 편집 ✅ (기본)
- **인라인 편집 모드 특징**:
  - 노드 헤더에 현재 에디터 타입 라벨 표시 (📝 Markdown Editor, ✨ Rich Editor, 📄 Text Editor) ✅
  - 인라인 편집 중에는 header 도구 상자(툴바)가 표시되지 않음 ✅
  - 편집 버튼은 기본적으로 숨김 처리 (도구 상자 토글으로 표시 가능) ✅

**도구 상자 편집** (도구 상자 버튼 클릭):
- 도구 상자의 "Rich" 또는 "Markdown" 버튼 클릭 시 → 해당 모드의 모달 편집 ✅
- 도구 상자의 "편집 버튼" 토글 버튼으로 노드의 편집 버튼 표시/숨김 제어 ✅

**선택 해제 및 초기화**:
- **캔버스 클릭**: 배경 클릭 시 모든 선택 및 편집 상태 초기화 ✅
- **Escape 키**: 편집 중이 아닐 때 Escape 키로 모든 선택 상태 초기화 ✅
  - 편집 중일 때는 각 에디터가 자체적으로 Escape 처리 (편집 취소)

**새 노드 생성**:
- 기본 `contentType`: `richeditor`
- 더블 클릭 시 → **Rich Editor 인라인 편집** 시작 (팝업 없음)

### 권장 사항
- **기본 모드**: Rich Editor ⭐
- **빠른 전환**: 도구 상자 아이콘으로 즉시 전환
- **직관적 UX**: 아이콘 클릭으로 바로 편집 시작

## 📐 구현 아키텍처

### 컴포넌트 구조

```
MindMapCanvas
  ├─ Canvas Toolbar (도구 상자) ✅
  │   ├─ Add Node 버튼 ✅
  │   ├─ Edit 버튼 ✅
  │   ├─ 📝 Rich Editor 아이콘 ✅ (모달 편집)
  │   └─ 📄 Markdown Editor 아이콘 ✅ (모달 편집)
  │
  └─ Node (렌더링) ✅
      ├─ Node Header (Text 모드일 때) ✅
      │   ├─ 📝 Markdown 버튼 ✅
      │   └─ ✏️ Rich 버튼 ✅
      │
      ├─ Edit Button (노드 선택 시) ✅
      │   └─ 편집 버튼 클릭 시 → 모달 편집 ✅
      │
      └─ Edit Mode Handler ✅
          ├─ 노드 클릭 → 편집 버튼 표시 ✅
          ├─ 편집 버튼 클릭 시 (모달 편집):
          │   ├─ RichEditorModal (오버레이) ✅
          │   └─ MarkdownEditorModal (오버레이) ✅
          │
          ├─ 더블 클릭 시 (인라인 편집):
          │   ├─ TextEditor (인라인) ← text 모드 ✅
          │   ├─ RichEditor (인라인) ← richeditor 모드 ✅
          │   └─ NodeEditor (인라인) ← markdown 모드 ✅
          │
          └─ 도구 상자 버튼 클릭 시 (모달 편집):
              ├─ RichEditorModal (오버레이) ✅
              └─ MarkdownEditorModal (오버레이) ✅
```

2. 노드 더블 클릭

   - Header 도구 상자(툴바)는 표시되지 않음 (더 넓은 편집 영역)
   - 편집 버튼은 기본적으로 숨김 처리
4. 직접 서식 적용 (Bold, Italic 등 - 키보드 단축키 사용)

6. 편집 종료 후 노드에 업데이트된 내용 표시

2. 도구 상자의 "📝 Rich" 버튼 클릭
3. **Rich Editor 모달** 열림
4. 큰 오버레이 모달에서 WYSIWYG 편집
5. 서식 적용 (Bold, Italic, Link 등)
6. "저장" 버튼 클릭 또는 `Alt+Enter`
7. 모달 닫히고 노드에 업데이트된 내용 표시

5. `Enter`로 저장, `Esc`로 취소
6. 노드 헤더의 "Markdown" 또는 "Rich" 버튼으로 모드 전환 가능

## 🔍 기술적 고려사항

### 1. 모달 렌더링

- **React Portal**: 모달을 body에 직접 렌더링하여 z-index 문제 방지
- **포커스 관리**: 모달 열릴 때 자동 포커스, 닫을 때 원래 요소로 포커스 복귀
- **배경 클릭**: 배경 클릭 시 취소 또는 확인 모달

### 2. 상태 동기화

- **편집 중 저장**: 모달에서 편집한 내용이 노드에 즉시 반영
- **취소 처리**: 취소 시 변경사항 무시
- **최적화 업데이트**: 큰 텍스트 편집 시 성능 최적화

### 3. 키보드 접근성

- **Tab 순서**: 모달 내 포커스 순서 관리
- **Esc 키**: 모달 닫기
- **단축키**: 일관된 단축키 체계

### 4. 모바일 대응

- **반응형 크기**: 모바일에서는 전체 화면 또는 작은 모달
- **터치 최적화**: 터치 친화적 UI
