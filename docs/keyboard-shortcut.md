## 캔버스 조작

### 전체 선택 해제 및 상태 초기화
- **`Esc`**: 모든 선택 상태 초기화
  - 노드/엣지 선택 해제
  - 편집 모드 취소
  - 팬닝 취소
  - 연결 모드 취소
  - 리사이즈 취소

> **참고**: 노드 편집 중일 때는 `Esc`가 편집 취소로 동작하며, 편집이 아닐 때만 전체 선택 해제로 동작합니다.

## 노드 선택 및 조작

### 노드 추가
- **`a`** 또는 **`A`**: 새 노드 추가
  - **한글/영어 입력 상태와 무관하게 동작** (`e.code` 사용)
  - 선택된 노드가 있으면 그 노드의 오른쪽 아래에 생성
  - 선택된 노드가 없으면 캔버스 중앙에 생성
  - 중앙에 노드가 여러 개 있으면 나선형으로 배치하여 겹치지 않게 함
  - 새로 추가된 노드가 자동으로 선택됨
  - 편집 중일 때는 동작하지 않음
  - 텍스트 입력 중일 때는 동작하지 않음 (텍스트 입력 우선)
  - IME 입력 중일 때는 동작하지 않음 (한글/중국어/일본어 입력 중)

### 편집 모드 진입
- **`e`** 또는 **`E`**: 선택된 노드 편집 모드 진입
  - **한글/영어 입력 상태와 무관하게 동작** (`e.code` 사용)
  - 노드가 선택되어 있을 때만 동작
  - 노드의 `contentType`에 따라 적절한 편집 모드 자동 선택:
    - `richeditor` (또는 미지정): Rich Editor 모달 편집
    - `markdown`: Markdown Editor 모달 편집
    - `text`: Text Editor 인라인 편집
  - 편집 모드 진입 후 키보드 포커스가 에디터로 이동
  - IME 입력 중일 때는 동작하지 않음 (한글/중국어/일본어 입력 중)

### 노드 삭제
- **`d`** 또는 **`D`**: 선택된 노드/엣지 삭제 (추천)
  - **한글/영어 입력 상태와 무관하게 동작** (`e.code` 사용)
  - 노드가 선택되어 있을 때만 동작
  - 연결된 엣지도 함께 삭제됨
  - 삭제 후 선택 상태 자동 해제
  - 확인 다이얼로그 표시
  - **주의**: 삭제된 노드는 되돌릴 수 없습니다 (Undo 미지원)
- **`Delete`** 또는 **`Backspace`**: 선택된 노드/엣지 삭제 (기존 방식)
  - 노드가 선택되어 있을 때만 동작
  - 연결된 엣지도 함께 삭제됨
  - 삭제 후 선택 상태 자동 해제
  - 확인 다이얼로그 표시

### 노드 복제 (계획됨)
- **`Ctrl/Cmd + D`**: 선택된 노드 복제
  - 복제된 노드는 원본 노드의 오른쪽에 배치
  - 복제된 노드가 자동 선택됨

### 노드 이동 (계획됨)
- **`↑` / `↓` / `←` / `→`**: 선택된 노드 이동
  - 화살표 키로 노드를 픽셀 단위로 이동
  - **`Shift + ↑/↓/←/→`**: 더 큰 단위로 이동 (10픽셀)

### 빠른 노드 추가 워크플로우
1. **`a`** 키 입력 → 새 노드 추가
2. 자동으로 선택된 새 노드에서 **`e`** 키 입력 → 편집 모드 진입
3. 내용 수정
4. **`Alt + Enter`** → 저장 후 종료

### 빠른 편집 워크플로우
1. 노드 클릭 → 노드 선택
2. **`e`** 키 입력 → 편집 모드 진입
3. 내용 수정
4. **`Alt + Enter`** → 저장 후 종료

## 노드 편집

### 인라인 편집 모드 (더블 클릭)

노드를 더블 클릭하면 현재 노드 타입에 맞는 인라인 에디터가 열립니다.

#### 공통 단축키 (모든 인라인 에디터)
- **`Alt + Enter`** (Mac: `Option + Enter`): 저장 후 편집 종료
- **`Ctrl/Cmd + Enter`**: 저장 (하위 호환성)
- **`Esc`**: 편집 취소 (변경사항 저장 안 됨)

---

## 리치 에디터 (Rich Editor)

### 인라인 편집 모드 (`RichEditor`)

노드의 `contentType`이 `richeditor`일 때 더블 클릭하면 인라인 리치 에디터가 열립니다.

#### 저장 및 종료
- **`Alt + Enter`**: 저장 후 편집 종료
- **`Ctrl/Cmd + Enter`**: 저장 (하위 호환성)
- **`Esc`**: 편집 취소

#### 일반 입력
- **`Enter`**: 새 줄 생성 (일반 텍스트)
- **`Shift + Enter`**: 새 줄 생성 (강제)
- 리스트 항목 내에서 **`Enter`**: 새 리스트 항목 생성

---

### 모달 편집 모드 (`RichEditorModal`)

노드의 편집 버튼을 클릭하거나 도구 상자에서 "Rich Editor" 버튼을 클릭하면 모달 편집창이 열립니다.

#### 저장 및 종료
- **`Alt + Enter`**: 저장 후 편집 종료
- **`Ctrl/Cmd + Enter`**: 저장
- **`Esc`**: 편집 취소

#### 텍스트 포맷팅
- **`Ctrl/Cmd + B`**: **Bold** (굵게)
- **`Ctrl/Cmd + I`**: *Italic* (기울임)
- **`Ctrl/Cmd + U`**: <u>Underline</u> (밑줄)
- **`Ctrl/Cmd + K`**: 링크 삽입/편집

#### 붙여넣기
- **`Ctrl/Cmd + V`**: 서식 있는 텍스트 붙여넣기 (HTML)
- **`Shift + Ctrl/Cmd + V`**: 플레인 텍스트만 붙여넣기 (서식 제거)

#### 포커스 관리
- **`Tab`**: 모달 내 다음 포커스 가능 요소로 이동
- **`Shift + Tab`**: 모달 내 이전 포커스 가능 요소로 이동

> **팁**: 모달은 포커스 트랩(Focus Trap) 기능을 제공합니다. `Tab` 키로 마지막 요소에 도달하면 첫 번째 요소로 순환 이동합니다.

## 마크다운 에디터

### 인라인 편집 모드 (`NodeEditor`)

노드의 `contentType`이 `markdown`일 때 더블 클릭하면 인라인 마크다운 에디터가 열립니다.

#### 저장 및 종료
- **`Alt + Enter`**: 저장 후 편집 종료
- **`Ctrl/Cmd + Enter`**: 저장 (하위 호환성)
- **`Esc`**: 편집 취소

#### 마크다운 입력
- **`Enter`**: 새 줄 생성
- **`Tab`**: 2칸 들여쓰기 삽입 (마크다운 컨벤션)
- **`Shift + Tab`**: 일반 동작 (에디터에서는 들여쓰기 제거 미지원)

---

### 모달 편집 모드 (`MarkdownEditorModal`)

노드의 편집 버튼을 클릭하거나 도구 상자에서 "Markdown" 버튼을 클릭하면 모달 편집창이 열립니다.

#### 저장 및 종료
- **`Alt + Enter`**: 저장 후 편집 종료
- **`Ctrl/Cmd + Enter`**: 저장
- **`Esc`**: 편집 취소

#### 마크다운 입력
- **`Enter`**: 새 줄 생성
- **`Tab`**: 2칸 들여쓰기 삽입
- **`Shift + Tab`**: 일반 동작

#### 포커스 관리
- **`Tab`**: 모달 내 다음 포커스 가능 요소로 이동
- **`Shift + Tab`**: 모달 내 이전 포커스 가능 요소로 이동

## 텍스트 에디터

### 인라인 편집 모드 (`TextEditor`)

노드의 `contentType`이 없거나 `text`일 때 더블 클릭하면 인라인 텍스트 에디터가 열립니다.

#### 저장 및 종료
- **`Enter`**: 
  - 단일 줄 입력 필드: 저장 후 편집 종료
  - 다중 줄 텍스트 영역: 새 줄 생성 (저장 안 됨)
- **`Alt + Enter`**: 저장 후 편집 종료 (텍스트 영역에서)
- **`Esc`**: 편집 취소

#### 자동 감지
- 텍스트가 짧거나 줄바꿈이 없으면 단일 줄 `input` 사용
- 텍스트가 길거나 줄바꿈이 있으면 다중 줄 `textarea` 사용

## 이미지 붙여넣기

### 캔버스에 이미지 붙여넣기

캔버스가 포커스된 상태에서 클립보드에 이미지가 있을 때:

- **`Ctrl/Cmd + V`**: 이미지 붙여넣기
  - 이미지를 자동으로 업로드
  - 새 이미지 노드 생성
  - 마우스 위치 또는 캔버스 중심에 배치

> **주의사항**:
> - 노드나 엣지를 편집 중일 때는 붙여넣기가 무시됩니다 (텍스트 편집 우선)
> - 텍스트 입력 필드(`input`, `textarea`)가 포커스되어 있을 때는 텍스트 붙여넣기가 우선됩니다
> - 지원 형식: PNG, JPEG, GIF, WebP 등 이미지 형식
> - 크기 제한: 10MB (API 설정에 따라 다를 수 있음)

## 모달 포커스 관리

모달 편집기(`RichEditorModal`, `MarkdownEditorModal`)는 접근성을 위한 포커스 트랩 기능을 제공합니다.

### 포커스 트랩
- **`Tab`**: 모달 내 다음 포커스 가능 요소로 이동
- **`Shift + Tab`**: 모달 내 이전 포커스 가능 요소로 이동
- 마지막 요소에서 `Tab`을 누르면 첫 번째 요소로 순환
- 첫 번째 요소에서 `Shift + Tab`을 누르면 마지막 요소로 순환

### 포커스 가능한 요소
- 편집 영역 (`contentEditable`, `textarea`)
- 모든 버튼 (저장, 취소, 포맷팅 버튼 등)
- 링크 및 입력 필드

## 플랫폼별 차이점

### Mac (macOS)
- **`Alt`** = **`Option`** 키
- **`Ctrl/Cmd`** = **`Command (⌘)`** 키
- **`Alt + Enter`** = **`Option + Enter`**

### Windows / Linux
- **`Alt`** = **`Alt`** 키
- **`Ctrl/Cmd`** = **`Ctrl`** 키
- **`Alt + Enter`** = **`Alt + Enter`**

## 사용 팁

### 빠른 편집 워크플로우
1. 노드 더블 클릭 → 인라인 편집 모드 진입
2. 내용 입력/수정
3. **`Alt + Enter`** → 저장 후 자동 종료

### 모달 편집 사용 시
1. 노드 선택 후 편집 버튼 클릭 → 모달 편집창 열기
2. 또는 도구 상자의 "Rich Editor" / "Markdown" 버튼 클릭
3. 편집 후 **`Alt + Enter`** 또는 저장 버튼 클릭

### 서식 있는 텍스트 붙여넣기
- 외부에서 복사한 서식 있는 텍스트를 리치 에디터에 붙여넣으면 서식이 유지됩니다
- 서식을 제거하고 싶으면 **`Shift + Ctrl/Cmd + V`** 사용

### 마크다운 코드 블록
마크다운 에디터에서 코드 블록을 작성할 때:
```markdown
`Tab` 키를 사용하여 들여쓰기 (2칸)
```

### 취소 및 되돌리기
- **`Esc`** 키를 누르면 현재 작업을 취소하고 변경사항을 저장하지 않습니다
- 편집 중 실수로 변경한 내용을 되돌리려면 **`Esc`**를 누른 후 다시 편집을 시작하세요

## 구현 가이드

### `e` 키: 편집 모드 진입

**현재 상태**: 구현 필요

**구현 위치**: `frontend/src/components/MindMapCanvas.tsx`

**구현 방법**:
```typescript
const handleKeyDown = (e: KeyboardEvent) => {
  // 기존 Escape 처리...
  
  // 'e' 키 처리: 선택된 노드 편집 모드 진입
  if ((e.key === 'e' || e.key === 'E') && selectedNodeId && !editingNodeId && !editingEdgeId) {
    e.preventDefault();
    const node = map?.nodes.find((n) => n.id === selectedNodeId);
    if (node) {
      // contentType에 따라 편집 모드 설정
      if (node.contentType === 'markdown') {
        setEditorMode('markdown');
      } else {
        setEditorMode('richeditor'); // 기본값 또는 richeditor
      }
      setEditingNodeId(selectedNodeId);
    }
  }
  
  // Delete/Backspace 키 처리: 선택된 노드/엣지 삭제
  if ((e.key === 'Delete' || e.key === 'Backspace') && !editingNodeId && !editingEdgeId) {
    e.preventDefault();
    
    // 입력 필드에서 입력 중일 때는 삭제 동작 막기
    const activeElement = document.activeElement;
    if (activeElement && (
      activeElement.tagName === 'INPUT' ||
      activeElement.tagName === 'TEXTAREA' ||
      activeElement.getAttribute('contenteditable') === 'true'
    )) {
      return; // 텍스트 입력 중일 때는 노드 삭제 안 함
    }
    
    if (selectedNodeId) {
      if (confirm(`노드를 삭제하시겠습니까?`)) {
        deleteNode(selectedNodeId);
        selectNode(null);
      }
    } else if (selectedEdgeId) {
      if (confirm(`엣지를 삭제하시겠습니까?`)) {
        deleteEdge(selectedEdgeId);
        selectEdge(null);
      }
    }
  }
};
```

### Delete/Backspace 키: 삭제

**현재 상태**: 구현 필요

**주의사항**:
- 텍스트 입력 필드(`input`, `textarea`, `contentEditable`)가 포커스되어 있을 때는 노드 삭제가 동작하지 않아야 합니다
- 삭제 전 확인 다이얼로그를 표시하는 것을 권장합니다
- 삭제 후 선택 상태를 자동으로 해제해야 합니다

## 주의사항 및 제한사항

### IME(Input Method Editor) 입력 지원

키보드 단축키는 **한글/영어 입력 상태와 무관하게 동작**하도록 구현되었습니다:

- **`e.code` 속성 사용**: 물리적 키 위치를 감지하여 언어 입력 상태와 무관하게 동작
- **`e.isComposing` 체크**: IME 조합 중(한글/중국어/일본어 입력 중)일 때는 단축키 무시
- **동작 예시**:
  - 한글 입력 모드에서 `a` 키 입력 → 노드 추가 (한글 "ㅁ"이 입력되지 않음)
  - 한글 입력 모드에서 `e` 키 입력 → 편집 모드 진입 (한글 "ㄷ"이 입력되지 않음)
  - 한글 입력 조합 중(`isComposing === true`) → 단축키 무시 (한글 입력 완료 대기)

### 키보드 단축키 충돌 방지

다음 상황에서는 키보드 단축키가 동작하지 않습니다:

1. **텍스트 입력 중**: `input`, `textarea`, `contentEditable` 요소에 포커스가 있을 때
   - 이유: 사용자가 텍스트를 입력하는 중이므로 노드 삭제 같은 조작을 방지
   - 예외: `Esc` 키는 항상 동작 (편집 취소)

2. **IME 입력 중**: `e.isComposing === true`일 때
   - 이유: 한글/중국어/일본어 입력 조합 중에는 단축키를 무시하여 자연스러운 입력 보장
   - 예: 한글 입력 중 "가나다"를 입력할 때 중간에 단축키가 작동하지 않음

3. **모달 열림**: 모달 다이얼로그가 열려 있을 때
   - 이유: 모달 내부의 입력 필드와 충돌 방지

4. **읽기 전용 모드**: `isReadOnly={true}`일 때
   - 이유: 읽기 전용 모드에서는 편집 불가

### 키보드 단축키 우선순위

1. **에디터 내부 단축키** (최우선)
   - `Alt + Enter`, `Ctrl/Cmd + Enter`, `Esc` 등
2. **노드 조작 단축키**
   - `e`, `Delete`, `Backspace`
3. **캔버스 조작 단축키**
   - `Esc` (선택 해제)
