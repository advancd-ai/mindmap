# 🖼️ Image Copy and Paste 기획서

## 🎯 목표

**클립보드에 복사된 이미지(스크린샷, 이미지 파일 등)를 붙여넣기만으로 쉽게 이미지 노드를 생성**할 수 있도록 하여, 사용자의 작업 흐름을 방해하지 않고 빠르게 시각적 콘텐츠를 마인드맵에 추가할 수 있게 한다.

## 📊 현재 상태

### 현재 구현
- **이미지 노드**: `embedUrl`과 `embedType: 'image'`로 이미지 표시 지원 ✅
- **이미지 업로드**: `EmbedDialog`를 통해 파일 선택하여 업로드 가능 ✅
- **업로드 API**: `POST /upload` 엔드포인트로 이미지 업로드 지원 ✅
- **제한사항**:
  - 이미지 노드 생성이 여러 단계 필요 (노드 생성 → Embed 메뉴 선택 → 파일 선택 → 업로드)
  - 클립보드에서 직접 붙여넣기 불가
  - 스크린샷이나 복사한 이미지를 빠르게 추가하기 어려움

## 🎨 요구사항

### 1. 기능 요구사항

#### 기본 이미지 붙여넣기
- [ ] **클립보드 감지**: 캔버스에서 `Ctrl/Cmd + V` 시 클립보드 이미지 감지
- [ ] **자동 노드 생성**: 이미지가 클립보드에 있으면 자동으로 이미지 노드 생성
- [ ] **자동 업로드**: 클립보드 이미지를 자동으로 서버에 업로드
- [ ] **위치 자동 설정**: 붙여넣기 위치에 노드 생성 (마우스 커서 위치 또는 캔버스 중심)

#### 지원되는 이미지 형식
- [ ] **스크린샷**: OS 기본 스크린샷 도구 (Mac: Cmd+Shift+4, Windows: Win+Shift+S)
- [ ] **이미지 복사**: 브라우저나 앱에서 이미지 복사 (Ctrl/Cmd+C)
- [ ] **파일 드래그 앤 드롭**: 이미지 파일을 캔버스에 드롭
- [ ] **이미지 형식**: PNG, JPEG, GIF, WebP, SVG, BMP

#### 사용자 경험
- [ ] **시각적 피드백**: 붙여넣기 시작 시 로딩 인디케이터
- [ ] **에러 처리**: 지원하지 않는 형식 또는 업로드 실패 시 사용자 친화적 에러 메시지
- [ ] **다중 이미지 지원**: 여러 이미지를 한 번에 붙여넣기 (선택사항)
- [ ] **노드 크기 자동 조정**: 이미지 비율에 맞춰 노드 크기 자동 조정

### 2. 기술 요구사항

#### 데이터 구조
```typescript
interface Node {
  id: string;
  label: string;              // 이미지 설명 또는 파일명
  embedUrl: string;           // 업로드된 이미지 URL
  embedType: 'image';         // 이미지 타입
  w: number;                  // 노드 너비 (이미지 크기에 맞춤)
  h: number;                  // 노드 높이 (이미지 크기에 맞춤)
  // ... 기존 필드
}
```

#### 클립보드 API
- **Clipboard API**: `navigator.clipboard.read()` 또는 `ClipboardEvent.clipboardData`
- **파일 처리**: `Blob` → `File` 변환
- **이미지 메타데이터**: 크기, 형식 정보 추출

### 3. UI/UX 요구사항

#### 붙여넣기 흐름
1. 사용자가 이미지를 클립보드에 복사
2. 캔버스에서 `Ctrl/Cmd + V` 입력
3. 시스템이 클립보드 이미지 감지
4. 자동으로 업로드 진행 (로딩 인디케이터 표시)
5. 업로드 완료 후 이미지 노드 생성 및 표시

#### 시각적 피드백
- **로딩 상태**: 업로드 중일 때 노드 위치에 로딩 스피너 표시
- **성공 표시**: 노드 생성 후 간단한 토스트 메시지 (선택사항)
- **에러 표시**: 업로드 실패 시 에러 메시지 표시

#### 키보드 단축키
- `Ctrl/Cmd + V`: 이미지 붙여넣기 (기존 텍스트 붙여넣기와 구분 필요)
- `Escape`: 업로드 취소 (로딩 중일 때)

```typescript
    filename: string,
    size: number,

  }
```

## 🔍 기술적 고려사항

### 1. 클립보드 접근 권한
- **보안 정책**: HTTPS 환경에서만 Clipboard API 사용 가능
- **사용자 권한**: 일부 브라우저는 사용자 인터랙션 필요
- **Fallback**: `ClipboardEvent.clipboardData` 사용 (이벤트 핸들러 내에서만 동작)

### 2. 이미지 형식 검증
- **MIME 타입 확인**: `file.type` 또는 `clipboardData.items[].type`
- **파일 확장자**: 이미지 확장자 검증
- **파일 크기**: 최대 크기 제한 (현재 10MB)

### 3. 업로드 최적화
- **비동기 처리**: 업로드는 비동기로 처리하여 UI 블로킹 방지
- **에러 핸들링**: 네트워크 오류, 파일 크기 초과 등 처리
- **진행률 표시**: 대용량 이미지 업로드 시 진행률 표시 (선택사항)

### 4. 노드 위치 계산
- **마우스 커서 위치**: Paste 이벤트 발생 위치
- **캔버스 중심**: 마우스 위치 정보가 없으면 캔버스 중심
- **충돌 방지**: 기존 노드와 겹치지 않도록 위치 조정

### 5. 이미지 크기 조정
- **원본 크기 유지**: 기본적으로 원본 크기 유지
- **노드 크기 제한**: 최대/최소 노드 크기 제한
- **비율 유지**: 이미지 비율에 맞춰 노드 크기 조정

## 🎓 사용자 가이드 (초안)

### 이미지 붙여넣기 기본 사용법

#### 방법 1: 스크린샷 붙여넣기
1. 스크린샷을 찍습니다 (Mac: `Cmd+Shift+4`, Windows: `Win+Shift+S`)
2. 마인드맵 캔버스를 클릭하여 포커스 설정
3. `Ctrl/Cmd + V`를 눌러 붙여넣기
4. 자동으로 이미지가 업로드되고 노드가 생성됩니다

#### 방법 2: 이미지 복사 붙여넣기
1. 브라우저나 앱에서 이미지를 선택하고 복사 (`Ctrl/Cmd + C`)
2. 마인드맵 캔버스를 클릭하여 포커스 설정
3. `Ctrl/Cmd + V`를 눌러 붙여넣기
4. 자동으로 이미지가 업로드되고 노드가 생성됩니다

#### 방법 3: 파일 드래그 앤 드롭 (Phase 3)
1. 이미지 파일을 찾습니다
2. 파일을 마인드맵 캔버스로 드래그
3. 놓으면 자동으로 업로드되고 노드가 생성됩니다

### 지원되는 이미지 형식
- PNG, JPEG/JPG, GIF, WebP, SVG, BMP
- 최대 파일 크기: 10MB

### 주의사항
- HTTPS 환경에서만 클립보드 접근 가능
- 일부 브라우저는 사용자 인터랙션(클릭 등) 후에만 클립보드 접근 허용
- 업로드 중에는 네트워크 연결이 필요합니다
